## OSI(Open System Interconnection) 7계층

 Q1. OSI 7계층이란 무엇인가(네트워크 계층에 대한 표준)    
 데이터는 네트워크를 통해서 상대방에게 전송이 된다. 이러한 네트워크를 통해 통신이 일어나는 과정을 7단계로 나눈것을 말한다 
 
 Q2. OSI 7계층을 나눈 이유는 무엇인가     
 7단계중에서 특정한 곳에 이상이 생기면 다른 단계의 장비나 소프트웨어를 건들지 않고 이상이 생긴 단계에서 고칠 수 있기 때문이다

### OSI 7계층의 역할
> 응용계층    
전송단위 : data, 프로토콜: HTTP, SMTP, POP3, DNS, SSH등   
역할 : 응용프로그램과 연계하여 사용자에게 편리한 환경을 제공해준다    

> 표현계층     
전송단위 : data   
프로토콜 : SMB,,,  
역할 : 데이터의 번역 및 암호화    

> 세션계층   
전송단위 : data   
프로토콜 : SSL, TLS   
역할 : 응용 프로그램들 간의 통신 관리 및 동기화 유지, 응용 프로그램들 사이의 접속을 설정/유지하며 데이터 전송시 동기점을 제공해서 오류 발생시 데이터를 재전송하거나 복구 할 수 있는 계층이다   

> 전송계층
전송단위 : 세그먼트   
프로토콜 : TCP, UDP   
역할 : 송신 측과 수신측의 연결 설정 및 유지, 데이터 전송에 관한 서비스를 제공하는 계층으로 송/수신 측의 실질적인 연결을 설정하여 신뢰성 있는 통신이 가능하도록 해준다   

> 네트워크 계층    
전송단위 : 패킷   
프로토콜 : IP, ICMP, ARP   
관련장치 : 라우터   
역할 : 데이터를 전송할때 경로를 제어해주고 최적의 경로를 선택한다. 수신호스트에 도착하기 위해 여러 시스템을 거치는데 이때 올바른 경로를 선택할 수 있도록 라우팅 기능을 해준며 ip를 정한다

> 데이터링크 계층    
전송단위 : 프레임   
프로토콜 : 토큰링, Ethernet   
관련장치 : 브리지, 스위치   
역할 : MAC주소를 사용하며 신뢰성 있는 정보를 전송하기 위한 역할을 수행한다. 네트워크 계층에서 받은 데이터를 프레임이라는 단위로 구성하여 필요한 정보를 추가한 뒤 물리계층으로 전달한다.    

> 물리 계층   
전송단위 : bits   
프로토콜 : rs-232,,   
관련장치 : 허브, 리피터    
역할 : 전송 매체를 통한 데이터 전송 및 데이터의 신호변환, 실제로 데이터를 전송하는 부분이다.    
 
### EX ) HI라는 메시지를 카톡을 통해 친구에게 보낸다
HI라는 데이터는 응용계층에서 헤더에 데이터와 함께 붙여 하위 계층으로 넘겨준다. 표현 계층에서 데이터를 받고 이를 암호화하여 헤더에 붙인 후 하위 계층으로 넘겨준다. 세션계층에서는 오류발생시의 데이터 재전송/복구를 위해 응용 프로그램들 사이의 동기점을 제공한다. 전송계층은 실질적으로 송/수신측의 연결을 설정하여 신뢰성 있는 통신이 될 수 있게 설정한다. 이후 네트워크 계층은 데이터가 올바른 경로로 보내질 수 있도록 라우팅 기능을 해준다. 이후 데이터 링크 계층에서 신뢰성 있는 정보를 전송하기 위해 필요한 정보를 추가하여 전송한다. 이후 물리계층에서 데이터를 수신자에게 bit 단위로 보내게 된다 . 이과정을 __Encapsulation__ 이라고 한다. 이후 수신자는 물리계층에서부터 응용계층까지 헤더에 있는 정보를 확인하며 상위 계층으로 데이터를 전달한다. 이 과정을 __Decapsulation__ 이라고 한다

#### 캡슐화/비캡슐화(Encapsulation/Decapsulation)
송신 단계에서 7단계를 거쳐 각각의 PDU를 추가해 Data를 캡슐화 하는 방식을 Encapsulation이라고한다. 수신 단계에서 7단계에 거쳐 각각의 PDU를 제거하여 Data를 얻는 방식을 Decapsulation이라고 한다.

## 프로토콜 데이터 단위(PDU/Process Data Unit)

### PDU란 무엇인가?
프로토콜 데이터 단위(Protocol Data Unit)는 데이터 통신에서 상위 계층이 전달한 데이터에 붙이는 제어정보이다. 데이터 자체는 동일하지만 각각의 레이어를 거치면서 헤더 정보가 추가되면서 불리는 이름이 달라진다.    

<br>

### OSI 7 계층의 PDU에 대해 설명해라    
tcp/ip계층에 따른 PDU는 아래와 같이 정의된다   

- 응용계층(Client) : Data    
클라이언트가 보낼 data를 헤더에 담는다    

- 전송계층(TCP) : Segment     
세션 계층 데이터에 출발, 목적지의 포트 번호를 헤더에 담아서 캡슐화한다    
- 네트워크계층(IP) : Packet    
세그먼트 헤더에 출발, 목적지의 IP 정보를 헤더에 담아서 캡슐화한다    

- 데이터링크계층(DataLink) : Frame     
패킷 헤더에 출발, 목적지의 MAC 정보를 담고 트레일러에 CRC(오류검출비트)를 담아서 캡슐화한다    


>물리계층(Computer Hardware) : Bit -> bit는 pdu가 아닌 단지 신호의 흐름이다. 즉 pdu는 2,3,4계층을 생각하는 것이 옳다   


<br>

### 데이터 캡슐화란 무엇인가?
데이터를 주고받기 위해서는 수신처와 송신처의 주소 그리고 데이터 통신을 제어하기 위한 제어 데이터들이 필요하다.이러한 제어 데이터와 함께 데이터를 보내기 위해 필요한 것들이 통합된 상태를 PDU라고 한다. 이렇게 데이터를 덧붙여 PDU를 완성한 것을 캡슐화라고 한다


- Datagram : 데이터크램은 사용자의 순수한 메시지, 즉 메시지를 다르게 부르는 말이다
